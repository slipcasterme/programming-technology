##Анализ предметной области интернет-магазина обуви
#1. Введение
Тема проекта – интернет-магазин по продаже обуви.
Проблема – существующие площадки либо перегружены (маркетплейсы), либо ограничены одним брендом. Проект предлагает удобный специализированный каталог с быстрым оформлением заказа.

#2. Предметная область
Целевая аудитория – покупатели обуви (мужчины, женщины), предпочитающие онлайн-шопинг.
Основные сущности:

Товар (модель, цена, описание, фото, размеры)

Категория

Пользователь (опционально)

Корзина

Заказ

#3. Конкуренты и их недостатки
Аналоги: Wildberries, Lamoda, брендовые магазины.
Проблемы аналогов – сложная навигация, смешение категорий, отсутствие удобных фильтров именно для обуви. Наш проект фокусируется только на обуви, упрощая выбор.

#4. Цель и задачи
Цель – создать веб-сайт для продажи обуви с каталогом, корзиной и оформлением заказов.
Задачи:

Спроектировать БД.

Разработать бэкенд на Flask.

Сверстать шаблоны (Bootstrap).

Реализовать корзину (сессии) и заказы.

Добавить админку для управления товарами.

Протестировать и выложить код на GitHub.

#5. Требования
Основная функциональность:

Просмотр каталога и детальной страницы товара.

Добавление в корзину, управление количеством.

Оформление заказа с вводом имени, email, адреса.

Сохранение заказов в БД.

Дополнительно:

Адаптивный дизайн (Bootstrap).

БД SQLite.

Публичный репозиторий GitHub.

#6. Ресурсы
Время (MVP): около 2–3 недель (проектирование – 2 дня, разработка – 7 дней, вёрстка – 3 дня, тестирование – 2 дня).
Финансы: затраты минимальны (хостинг бесплатный или недорогой, лицензии бесплатны).
