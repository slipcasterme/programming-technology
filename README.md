# Анализ предметной области интернет-магазина обуви

### Введение

**Тема**: интернет-магазин по продаже обуви.

**Название**: StepStyle (рабочее название).

**Планируемые внешние интеграции**: платежные системы (ЮKassa или Stripe), API служб доставки (СДЭК, Почта России), Yandex Cloud для хранения изображений.

**Описание**: сервис представляет собой специализированный интернет-магазин обуви с удобным каталогом, корзиной и оформлением заказов. В отличие от крупных маркетплейсов, StepStyle фокусируется исключительно на обуви, предлагая продуманную систему фильтрации (размер, сезон, бренд, материал) и детальные карточки товаров. Магазин ориентирован на покупателей, которые ценят простой и быстрый выбор без лишнего шума.

---

### Предметная область

#### Целевая аудитория
- Мужчины и женщины от 18 до 45 лет, предпочитающие покупать обувь онлайн.
- Люди, которым неудобно пользоваться перегруженными маркетплейсами.
- Покупатели, ищущие конкретные модели, размеры или сезонную обувь.
- Владельцы небольших брендов обуви (как потенциальные партнёры).

#### Хранимые сущности

**Сущности и атрибуты**

| Сущность       | Атрибуты                                                                 |
|----------------|--------------------------------------------------------------------------|
| Товар          | Название, цена, описание, изображение(я), размеры, цвет, материал, сезон |
| Категория      | Название (кроссовки, ботинки, туфли и т.д.)                              |
| Бренд          | Название, логотип                                                        |
| Пользователь   | Имя, фамилия, email, пароль (хеш), телефон, адреса доставки              |
| Корзина        | Товары, количество, дата создания                                        |
| Заказ          | Пользователь, товары, сумма, статус, дата, способ доставки, способ оплаты|
| Статус заказа  | Название (новый, оплачен, отправлен, доставлен, отменён)                 |
| Отзыв          | Товар, пользователь, оценка, текст, дата                                 |

**Пример**

| Сущность       | Атрибуты                                                                 |
|----------------|--------------------------------------------------------------------------|
| Товар          | Кроссовки Nike Air Max, 12990 ₽, «Легендарная модель...», nike_air_max.jpg, 41–45, чёрный/белый, текстиль/кожа, лето |
| Категория      | Кроссовки                                                                |
| Бренд          | Nike, nike_logo.svg                                                      |

| Корзина        | Товар: Nike Air Max (2 шт.), дата: 2026-02-25                            |
| Заказ          | Иван Петров, товары: Nike Air Max (2 шт.), 25980 ₽, статус: «оплачен», 2026-02-25, СДЭК, карта онлайн |
| Отзыв          | Nike Air Max, Иван Петров, 5, «Отличные кроссовки, удобные», 2026-02-26  |

**Особенности**
- У одного товара может быть несколько изображений (разные ракурсы, цвета).
- Товар может относиться к нескольким категориям (например, «кроссовки» и «спортивная обувь»).
- Размерная сетка может различаться у разных брендов.
- Пользователь может иметь несколько адресов доставки.
- Корзина хранится в сессии для неавторизованных и привязывается к пользователю после входа.
- Заказ содержит снимок данных товара на момент покупки (цена, название).

### Проблема и текущее состояние

#### Конкуренты

**Wildberries / Ozon**  
Крупнейшие маркетплейсы с огромным ассортиментом.  
*Сильные стороны*: широчайший выбор, быстрая доставка, отзывы, программы лояльности.  
*Слабые стороны*: сложная навигация, много рекламы, товары разных категорий смешаны, фильтры по обуви ограничены, невозможно отследить историю покупок конкретной модели.

**Lamoda**  
Специализируется на одежде и обуви, есть примерка.  
*Сильные стороны*: удобный интерфейс, качественные фото, примерка перед покупкой.  
*Слабые стороны*: высокие цены, не всегда полный размерный ряд, ограниченный выбор бюджетных моделей.

**Брендовые магазины (Nike, Adidas, Zenden и др.)**  
Официальные сайты производителей.  
*Сильные стороны*: гарантия подлинности, новинки первыми.  
*Слабые стороны*: только один бренд, нет возможности сравнить с аналогами других марок.

**Специализированные обувные онлайн-магазины (Kari, Westfalika и др.)**  
*Сильные стороны*: фокус на обуви, удобные фильтры.  
*Слабые стороны*: устаревший дизайн, медленная работа, отсутствие современных функций (например, сохранение корзины между устройствами).

#### Существующие проблемы и недостатки
- Отсутствие сервиса, который объединяет удобный каталог, расширенные фильтры (по материалу, сезону, типу застёжки) и простой процесс заказа.
- Маркетплейсы перегружены информацией, не связанной с обувью.
- Невозможность сохранять историю покупок и желаемые модели в личном кабинете с привязкой к конкретному пользователю.
- Слабая адаптация под мобильные устройства у многих специализированных магазинов.
- Отсутствие единой платформы, где можно собрать обувь разных брендов, но с удобной обувной спецификой.

**StepStyle** решает эти проблемы, предлагая:
- Только обувь, без лишних категорий.
- Детальные фильтры, важные именно для обуви (полнота, материал подошвы, сезон).
- Личный кабинет с историей заказов и списком желаемого.
- Современный адаптивный дизайн.
- Возможность оставлять отзывы с фото.

### Цель и задачи проекта

**Целью** является проектирование и разработка специализированного интернет-магазина обуви с удобным каталогом, корзиной и системой заказов.

**Основные задачи**:
1. Проанализировать предметную область и определить требования.
2. Спроектировать базу данных и архитектуру приложения.
3. Разработать бэкенд на Flask с использованием SQLAlchemy.
4. Создать адаптивные шаблоны страниц (каталог, карточка товара, корзина, оформление заказа, личный кабинет).
5. Реализовать функционал корзины (сессии + база данных для авторизованных пользователей).
6. Интегрировать платежную систему и службы доставки.
7. Разработать админ-панель для управления товарами, заказами и пользователями.
8. Провести тестирование и развернуть проект на хостинге.

### Основные требования

#### Функциональные требования
- Просмотр каталога обуви с фильтрацией по категории, бренду, размеру, цвету, материалу, сезону, цене.
- Детальная страница товара с несколькими фото, описанием, выбором размера, добавлением в корзину.
- Регистрация и авторизация пользователей.
- Личный кабинет с историей заказов, списком желаемого (избранное), сохранёнными адресами.
- Работа с корзиной: добавление, изменение количества, удаление товаров.
- Оформление заказа: выбор адреса, способа доставки и оплаты.
- Интеграция с платёжными системами (ЮKassa) и службами доставки (СДЭК).
- Админ-панель: управление товарами (CRUD), просмотр и изменение статусов заказов, управление пользователями.
- Поиск товаров по названию и описанию.
- Возможность оставлять отзывы с оценкой и фото.
- Импорт/экспорт товаров (например, через CSV).

#### Нефункциональные требования
- Адаптивный дизайн (мобильные устройства, планшеты, десктоп).
- Минималистичный и современный интерфейс.
- Хеширование паролей (bcrypt).
- Хранение изображений в S3-совместимом хранилище (Yandex Cloud).
- Кэширование часто запрашиваемых данных (Redis).
- Сбор метрик (количество посещений, заказов, ошибок).
- Поддержка русского языка (интерфейс, валюты).
- Документация API для потенциальной интеграции с мобильным приложением.
- Использование PWA для мобильной версии (возможность установки на телефон).
- Соответствие требованиям безопасности (защита от XSS, CSRF, SQL-инъекций).
