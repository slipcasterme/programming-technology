### Сравнительный анализ инструментов разработки

Для реализации бэкенда интернет-магазина обуви необходимо выбрать веб-фреймворк. Рассмотрены три популярных инструмента на языке Python: Flask, Django и FastAPI.



###### **Выбранные инструменты для сравнения:**

1. Flask
2. Django
3. FastAPI



###### **Критерии сравнения:**

* Функциональность – набор встроенных возможностей (ORM, аутентификация, админка и т.д.).
* Удобство использования – простота старта, понятность для разработчика.
* Производительность – скорость обработки запросов.
* Гибкость – возможность расширения и выбора компонентов.
* Документация и сообщество – наличие качественной документации и поддержки.
* Совместимость – легкость интеграции с другими технологиями (SQLAlchemy, Celery, Redis).
* Безопасность – встроенные механизмы защиты от распространенных уязвимостей.

**Достоинства и недостатки**

###### Flask

Достоинства: легкий, гибкий, позволяет использовать только нужные компоненты, идеален для небольших и средних проектов.

Недостатки: многое нужно реализовывать вручную или подключать расширения, нет строгой структуры.



###### Django

Достоинства: мощная встроенная функциональность (админка, ORM, аутентификация), высокий уровень безопасности, быстрая разработка типовых функций.

Недостатки: избыточен для небольших проектов, менее гибкий, сложнее настройка под нестандартные задачи.



###### FastAPI

Достоинства: высокая производительность, асинхронность, автоматическая документация API, современный подход, встроенная валидация.

Недостатки: основной фокус на API, для классического сайта с шаблонами нужно добавлять дополнительные компоненты (Jinja2, статические файлы), сообщество меньше.



##### **Вывод**

Для проекта интернет-магазина обуви выбран Flask.



###### Обоснование:

1. Проект не требует сверхвысокой производительности, но важна гибкость и возможность быстрого прототипирования.
2. Flask позволяет подключать только необходимые компоненты (Flask-SQLAlchemy, Flask-Login, Flask-WTF), что соответствует уровню сложности «чуть выше простого».
3. Django избыточен для MVP, а FastAPI больше ориентирован на создание API и менее удобен для классического веб-сайта с шаблонами.
4. Flask имеет обширное сообщество и документацию, что важно для учебного проекта.



### Технологический стек проекта

На основе проведенного анализа и с учетом требований к проекту «StepStyle – интернет-магазин обуви» формируется следующий технологический стек.



###### **Бэкенд**

* Язык программирования: Python 3.10+
* Веб-фреймворк: Flask – выбран за гибкость и простоту.
* База данных: SQLite (на этапе MVP) с возможностью перехода на PostgreSQL при масштабировании.
* ORM: SQLAlchemy (через Flask-SQLAlchemy) для работы с БД.
* Аутентификация и авторизация: Flask-Login.
* Формы и валидация: Flask-WTF с WTForms.
* Миграции БД: Flask-Migrate (Alembic).
* Кэширование (опционально): Redis (для хранения сессий, кэша каталога).
* Фоновые задачи (опционально): Celery (для отправки email, обновления статусов заказов).
* Работа с изображениями: Flask-Uploads (или интеграция с облачным хранилищем).
* Логирование: стандартный logging с ротацией.



###### **Фронтенд**

* HTML-шаблоны: Jinja2 – встроенный шаблонизатор Flask.
* CSS-фреймворк: Bootstrap 5 – для быстрой адаптивной верстки.
* Дополнительные стили: собственный CSS для кастомизации.
* JavaScript: нативный JS для динамических элементов (фильтры, корзина без перезагрузки).
* AJAX: Fetch API для асинхронных запросов (фильтрация, обновление корзины).



###### **Хранение данных**

* База данных: SQLite (разработка), PostgreSQL (для продакшена).
* Хранилище изображений: Yandex Cloud Object Storage (S3-совместимое) или локальное хранение на этапе MVP.
* Резервное копирование: ежедневные дампы БД.
* Интеграции
* Платёжная система: ЮKassa (API для приема платежей).
* Службы доставки: API СДЭК (расчет стоимости и отслеживание).
* Отправка email: SMTP (Yandex или другой почтовый сервис).



###### **Инфраструктура и развертывание**

* Версионный контроль: Git, репозиторий на GitHub.
* Виртуальное окружение: venv или Poetry.
* Веб-сервер: Gunicorn (для продакшена).
* Прокси-сервер: Nginx (раздача статики, проксирование).
* Платформа хостинга: Яндекс.Облако / VPS (например, TimeWeb, Reg.ru) или бесплатный PythonAnywhere для демонстрации MVP.
* CI/CD (опционально): GitHub Actions для автоматического тестирования и деплоя.



###### **Инструменты разработки**

* IDE: PyCharm / VS Code.
* Тестирование: pytest, Flask-Testing.
* Документация: Sphinx (для API).
* Управление зависимостями: pip + requirements.txt.



###### **Соответствие требованиям**

* Адаптивность: Bootstrap 5 обеспечивает корректное отображение на мобильных устройствах.
* Безопасность: пароли хешируются (werkzeug.security или bcrypt), все формы защищены CSRF-токенами, используется HTTPS.
* Локализация: на этапе MVP – русский язык, в дальнейшем возможна поддержка английского через Flask-Babel.
* Производительность: использование кэша Redis, минификация статических файлов.



##### **Заключение**

Выбранный стек позволяет реализовать все функциональные требования проекта в рамках лабораторных работ, обеспечивая необходимую гибкость, простоту разработки и возможность последующего масштабирования.
